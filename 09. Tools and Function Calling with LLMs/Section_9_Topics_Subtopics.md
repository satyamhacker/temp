### Video---1 --- Topic--- Understanding Toolings in Large Language Models

* **[Introduction to Tooling]:** Tooling is introduced as a "great concept" that allows a large language model (LLM) to use external tools and libraries while performing actions. It gives the LLM "extra leverage or mileage" to search for external data or perform actions it was not originally trained for.
* **[The Limitation of Existing Training Data]:** The speaker provides an example prompt: "did Donald Trump won the 2024 presidential election". If asked directly via LangChain or a hyper terminal (using Ollama and the Qwen 2.5 model), the model will fail to answer correctly. It returns a response stating that its last update was in October 2023, so there is no official result for 2024.
* **[The Power of Tool Binding]:** By including a "tool binding" with a Wikipedia tool, the exact same query will yield a correct response: "yes, Donald Trump won the election in 2024". The LLM searches its available tools, finds context it previously lacked, summarizes the answer, and passes it back to the user.
* **[LangChain Definitions of Tools and Toolkits]:** On the LangChain website, tools are defined as "utilities designed to be called by the model." Their inputs are generated by models, and their outputs are passed back to the models. A "toolkit" is a collection of tools meant to be used together. Users are also encouraged to write their own custom tools.
* **[Existing Search Tools and their Outputs]:** LangChain offers built-in tools like the Bing search tool, Brave search tool, and DuckDuckGo search. Using their APIs, these search tools return data such as the URL, a text snippet, and the title. Some engines also support image or tabular searches.
* **[Installing and Using a Search Tool]:** Using the Brave search tool as an example, the speaker explains it acts like another library installed on top of the `langchain-community` package. After supplying API keys, the tool can be run and bound to the LLM. LangSmith can be used alongside this to monitor observability and traceability behind the scenes.
* **[Specialized Toolkits]:** Tooling includes more than just web search. The speaker highlights a "code interpreter tool" supporting Python, JavaScript, and Ruby. There are productivity tools like GitHub, GitLab, and Gmail toolkits. Additionally, a "Playwright browser toolkit" exists for web extraction and UI testing, along with tools for databases.

### Video---3 --- Topic--- Building Custom Tools

* **[LangChain Guidelines for Custom Tools]:** According to the LangChain tool page, when constructing an agent, a list of tools must be provided. Every tool must consist of specific components: a name, description, arg schema, and return direct.
* **[Three Methods to Create Tools]:** The documentation outlines three ways to create custom tools:
1. Standard Python functions.
2. LangChain Runnables (making an entire chain act as a tool).
3. Subclassing from `BaseTool`.


* **[Using the @tool Decorator]:** The easiest approach uses a decorator. The import is `from langchain.tools import tool`. By placing the `@tool` decorator above any Python function, it instantly becomes a LangChain tool.
* **[Creating the "add_numbers" Tool]:** A Python method `def add_numbers` is created, taking two integer parameters (`a` and `b`) and returning their sum as an integer. Crucially, a description property is added: "Add two numbers and return results". The speaker emphasizes that this description is vital because the LLM reads it to understand which tool it needs to call.
* **[Invoking Custom Tools]:** Because the custom function is now a LangChain runnable, it can be called directly. The speaker uses `add_numbers.invoke(a=10, b=20)` and prints the result, yielding `30`. They note that calling `.run` or `.invoke` achieves the exact same thing.
* **[Creating Additional Mathematical Tools]:** Using the exact same decorator method, two more tools are created: `subtract_numbers` (description: "Subtract two numbers and return result") and `multiply_numbers` (description: "Multiply two numbers").
* **[Structuring the List of Tools]:** All tools are placed into an array: `tools = [wikipedia, add_numbers, subtract_numbers, multiply_numbers]`. To store their metadata, a dictionary-like structure named `list_of_tools` is created by iterating `for tool in tools` and using `tool.name` to map the tools.
* **[Reviewing Tool Metadata]:** Printing `list_of_tools` reveals the internal structures. The Wikipedia tool is listed as `WikipediaQueryRun` showing settings like "top k result is 3" and "language is English". The custom math tools are recognized as `StructuredTool` objects, complete with their custom descriptions and argument schemas.

### Video---4 --- Topic--- Binding Tools to the Large Language Model

* **[The Objective of Binding]:** The goal is to map the array of tools to the LLM so the model can intelligently select the correct tool for a user's query rather than answering from its internal knowledge.
* **[Testing the Unbound Model]:** A baseline test is performed by invoking the LLM with "when did avatar movie released The Ways of Water". The model correctly answers "2022" from its own memory. However, it would fail if asked about Donald Trump's 2024 election. The goal is to force the model to search external tools instead.
* **[Tool Compatibility Limitations]:** The speaker warns that not all LLMs support toolings. For example, "DeepSeek R1" currently does not support toolings and will throw an error if binding is attempted. The Ollama website has a specific "tools" filter confirming that models like Qwen 2.5, Llama 3.1, and Llama 3.2 support tool binding.
* **[Executing the bind_tools Method]:** A new variable is created using the built-in LangChain method: `llm_with_tools = llm.bind_tools(tools)`.
* **[Testing Wikipedia Tool Selection]:** The bound model is invoked with the question "when did avatar movie got release". Instead of an actual answer, the output returns metadata under `tool_calls`. The AI smartly identified the "Wikipedia" tool and generated a query argument on the user's behalf: "Avatar the Way of Water release date".
* **[Observability via LangSmith]:** Checking the LangSmith trace for `ChatOllama` shows the exact functions/tools supported (Wikipedia, add_numbers, etc.). It displays the human input, the AI calling the Wikipedia tool with the generated query, and notes that there is "no YAML response" because it hasn't executed yet.
* **[Testing Custom Math Tool Selection]:** The query is changed to "what's the sum of 2 and 4". The LLM perfectly selects the `add_numbers` tool, passing arguments `A is 2 and B is 4`.
* **[Testing Inference Capabilities]:** The query is changed to "what is the double of 2". The LLM infers that "double" requires multiplication, selects the `multiply_numbers` tool, and passes the arguments `2` and `2`.
* **[The Missing Execution Step]:** The speaker points out a crucial gap: the LLM is correctly invoking the *selection* of the right tool and formatting arguments, but the actual execution result (e.g., the answer "4" or the year "2022") is nowhere to be found. A script to actively execute these tools must be written.

### Video---5 --- Topic--- Writing the Execution Logic for Custom Tools

* **[The Execution Workflow]:** The workflow involves the user's question moving through a prompt template to the model, which triggers tool calling. However, to return an actual answer (like "sum of two and four is six"), explicit logic must be written to execute the tool and feed the result back to the model.
* **[The Three-Message Prompt Structure]:** To make the LLM understand how to execute tools, a specific prompt template must be created holding three messages: the "Human Message" (user query), the "AI Message" (tool selected by the AI), and the "Tool Message" (execution logic output).
* **[Appending the Human Message]:** `HumanMessage` is imported from `langchain_core.messages`. An array named `messages` is created. A query ("did Donald Trump win the 2024 presidential election?") is wrapped in a `HumanMessage` and appended to the array.
* **[Appending the AI Message]:** The bound model is called via `llm_with_tools.invoke(query)` to generate the `ai_message`. This contains the tool call metadata (e.g., selecting Wikipedia). This `ai_message` is appended to the `messages` array.
* **[Extracting the Tool Name]:** To find which tool needs executing, a loop is created: `for tool_call in ai_message.tool_calls`. The tool name is extracted and lowercased (`.lower()`) "just in case, because maybe if the tool comes in capital letter from the LM, it could be a problem". This name is passed to `list_of_tools` to fetch the actual runnable tool.
* **[Invoking the Selected Tool]:** The extracted tool is executed using `tool_invoke = execute_tool.invoke(tool_call)`.
* **[Appending the Tool Message]:** The result of `tool_invoke` is appended to the `messages` array. (The speaker notes they accidentally appended the raw `tool_call` first, which made things messy, so they cleared the message array and correctly appended the `tool_invoke` result).
* **[Reviewing the Contextual Array]:** Printing the `messages` array now shows three distinct parts perfectly lined up: the human's query, the AI identifying Wikipedia, and the raw Wikipedia text execution detailing endorsements for the "2024 US presidential election". This provides the full context needed for the LLM.

### Video---6 --- Topic--- Finalizing Execution and Generating LLM Responses

* **[Passing the Complete Message Array]:** The complete `messages` array (containing the user query, AI tool selection, and raw tool output) is passed back to the LLM using `final_output = llm_with_tools.invoke(messages)`.
* **[Evaluating the Wikipedia Response]:** Printing `final_output.content` yields a perfectly formulated natural language response from the LLM: "Based on the information provided from Wikipedia, Donald Trump did win the 2024 presidential election." The local model, which previously had no internet access or 2024 data, successfully retrieved and answered using the tool. LangSmith trace confirms the Wikipedia tool executed flawlessly.
* **[Executing the Custom Addition Tool]:** The query is updated to "what is the sum of 2 and 22". Executing the entire code block invokes the `add_numbers` custom tool and returns: "The sum of 2 and 22 is 24."
* **[Executing the Custom Multiplication Tool]:** The query is updated to "what is the double of 2". Executing the code block successfully calls `multiply_numbers` and returns: "The double of 2 is 4."
* **[Final Takeaways]:** The speaker concludes that the user now understands how to enable a local LLM to bind and invoke both custom and community-built tools. They encourage the viewer to create their own structured tools and invoke them from within prompt templates for their own large language model applications.
